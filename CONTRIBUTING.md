# Contributing

Thanks for helping improve open-source projects by [Javan Rasokat](https://github.com/JavanXD).

## Before you open a PR

1. **Check for repo-specific docs** — many projects override this file. Look for `CONTRIBUTING.md` in the repository root first.
2. **Search existing issues and PRs** — avoid duplicate work.
3. **Security issues** — see [`SECURITY.md`](SECURITY.md). Do **not** file public issues for vulnerabilities.

## General workflow

1. Fork the repository (unless you have write access).
2. Create a focused branch: `fix/short-description` or `feat/short-description`.
3. Make a **small, scoped change** — one logical fix or feature per PR.
4. Match existing code style, naming, and documentation level in that repo.
5. Test locally when the project has tests or a runnable demo.
6. Open a pull request with a clear title and description.

## Pull request guidelines

- Explain **what** changed and **why**.
- Link related issues (`Fixes #123`) when applicable.
- Note breaking changes, migration steps, or new dependencies.
- Keep drive-by refactors out of unrelated PRs.

## Review expectations

- Maintainers review in spare time — please be patient.
- Feedback may request narrower scope, tests, or source verification (especially for data/catalog changes).
- Not every PR will be merged; issues are a good place to discuss ideas before coding.

## Code of conduct

This project follows the [Contributor Covenant](CODE_OF_CONDUCT.md). Be respectful and constructive.
